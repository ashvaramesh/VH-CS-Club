ImpactLink

ImpactLink is a community-focused app connecting high school and college students with volunteer, internship, and civic engagement opportunities. Designed to track impact through logged hours, badges, and achievements, it encourages users to actively participate in community service.

Overview

Setup Instructions:
1. Clone the Repository: git clone https://github.com/ashvaramesh/swift-project
2. Install Dependencies: Open the project in Xcode, navigate to File > Swift Packages > Add Package Dependency, and add Firebase’s GitHub link.
3. Configure Firebase: Set up a Firebase project, add your app, and download GoogleService-Info.plist to the project root.

Feature Overview:
- Opportunity Tracking (log hours, view stats, track achievements)
- Starred Activities (bookmark and manage specific opportunities)
- Profile Customization (add skills, set goals, choose avatars)
- Gamification (earn badges, appear on leaderboards)
- Dynamic Filters (browse by volunteer, government, environmental, etc.)
- Accessibility (screen reader and dark mode support)

Usage Instructions:
1. Onboarding: New users complete onboarding to set interests, pick an avatar, and get a quick tour.
2. Main Navigation: Access Home for opportunities, Profile for personal achievements, and Leaderboard for top volunteers.
3. Logging Hours: Use the Volunteer Tracker to add time and reflect in the journal.
4. Creating Events: Add new volunteer events in Create Event.

Future Enhancements:
- Real-Time Firebase Updates for immediate feedback across users
- Expanded Accessibility for improved navigation with screen readers

Development Timeline (9/23/24 - 11/15/24)

Starting on 9/23/24 (Monday), we expanded data models to include categories, badges, and improved starred activities. By 9/24/24 (Tuesday), the VolunteerTrackerView was implemented, and on 9/25/24 (Wednesday), badges and leaderboards began shaping the gamification aspect.

Through 9/26/24 (Thursday) - 9/30/24 (Monday), we added event creation, profile customization, starred activities, and onboarding flows. Enhancements continued into October, with dynamic filtering by 10/3/24 (Thursday), dark mode toggles by 10/4/24 (Friday), and impact tracking graphs by 10/5/24 (Saturday).

Early October (10/6 to 10/11) focused on accessibility enhancements, improved event creation logic, and bug fixes. Mid-October (10/12 to 10/23) involved dynamic leaderboard data, refined profile customization, advanced impact graph filters, and continuous accessibility improvements.

By late October into mid-November (10/24 to 11/15), we integrated Firebase for data storage and retrieval, implemented chat functionality using a ChatGPT-powered assistant (starting around 11/11/24), and polished the UI thoroughly. The final steps on 11/15/24 (Friday) included securing the API key and ensuring accessibility labels were consistently applied.

Key Milestones

  • 9/23/24: Data model expansions (Categories, Badge structs, StarredActivity structs, Impact structs)
  • 9/24/24: Implemented VolunteerTrackerView with CalendarView and ImpactGraphView
  • 9/25/24: Introduced BadgesView and LeaderboardsView, kickstarting the gamification layer
  • 9/30/24: Added OnboardingView, enhancing initial user experience
  • 10/3/24: Dynamic filtering in HomeView for category-based browsing
  • 10/4/24: Dark mode toggle in MoreView
  • 10/5/24: ImpactGraphView with bar charts to visualize weekly volunteer hours
  • 10/12/24: Leaderboards now update dynamically in real-time
  • 11/11/24: Chatbot integration planning began, introducing conversational guidance
  • 11/15/24: API key security, final UI polishing, and stable release preparations

Mentor’s Guidance Throughout Development

While the README timeline doesn’t explicitly mention the mentor, their influence was woven throughout the process. From the initial data model expansions on 9/23/24 to the introduction of complex features like the VolunteerTrackerView on 9/24/24, our mentor offered backend guidance, ensuring data structures and Firebase setup were optimized for real-time updates.

As UI features evolved (e.g., badges and leaderboards on 9/25/24, onboarding flows by 9/30/24, and dynamic filtering on 10/3/24), the mentor provided critical feedback on improving accessibility and navigation. Their continuous support helped refine color contrasts, button placements, and error handling—making the app more intuitive.

Even in later stages, like integrating the ChatGPT-powered assistant (post 11/11/24), the mentor’s guidance ensured the UI accommodated new interaction patterns gracefully. By 11/15/24, thanks in part to the mentor’s advice, the app offered a polished experience, from efficient backend queries to a seamless and accessible interface.

Resources & Research Utilized

We leveraged official SwiftUI documentation throughout development, ensuring adaptive and accessible interfaces from the very start (9/23/24 onwards). Firebase’s official guides and community forums were referenced extensively, especially around data synchronization (e.g., upon integrating VolunteerTrackerView on 9/24/24 and dynamic filtering on 10/3/24).

User feedback from beta testers, gathered after new features like badges (9/25/24) and dark mode toggling (10/4/24), guided iterative improvements. Continuous testing and user interviews shaped how we refined event creation, onboarding (9/30/24, 10/9/24, 10/10/24), and accessibility enhancements (10/6/24, 10/15/24).

External references included Apple’s Human Interface Guidelines to maintain UI consistency, OpenAI’s API documentation for the ChatGPT assistant integration (starting 11/11/24), and ongoing community discussions for best practices in state management and data modeling.

Evolution of the Design

Early design decisions after 9/23/24 focused on structuring categories and impacts visually, while 9/25/24 enhancements brought badges and leaderboards into a coherent UI. As profile customization rolled out by 9/27/24 and onboarding by 9/30/24, the UI evolved to accommodate guided steps, distinct icons, and accessible color schemes.

Dynamic filtering on 10/3/24, dark mode by 10/4/24, and impact graphs on 10/5/24 led us to refine layouts, ensure charts remained legible in all modes, and maintain a consistent style across views.

With continuous user testing (throughout October) and final touches in November, by 11/15/24 the app’s interface achieved a polished, intuitive design. The final product balanced functionality with visual appeal, clear navigation, and robust accessibility.
